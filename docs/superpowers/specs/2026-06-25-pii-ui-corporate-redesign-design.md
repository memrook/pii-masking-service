# Редизайн Web UI: корпоративный стиль «Прогресс Агро» (Вариант A)

**Дата:** 2026-06-25
**Файл:** `app/static/index.html` (единственный затрагиваемый файл)

## Цель

Переоформить Web UI сервиса PII-маскирования в корпоративный стиль «Прогресс Агро»
по образцу `~/ClaudeCode/курсы/Курс_сайт_v2.html`. Меняется **только разметка и стили**.
Вся функциональность и JS-логика сохраняются 1-в-1.

## Решения (зафиксированы с пользователем)

- **Брендинг:** полный бренд «Прогресс Агро» — логотип-лист (SVG из референса),
  шапка «Прогресс Агро · PII-маскирование».
- **Компоновка:** стики-шапка + компактный hero + рабочая область. Без длинного лендинга, без футера-секций.
- **Эстетика:** Вариант A «Воздушная корпоративная карточка» — максимально близко к референсу.

## Дизайн-язык (из референса)

- **Шрифт:** Inter через системный стек (`"Inter","Segoe UI",-apple-system,Roboto,Arial,sans-serif`),
  как в референсе — **без `@import`/CDN**. Текущий `<link>` на Google Fonts (Roboto) удаляется.
- **Палитра (CSS-переменные):**
  - `--green-900:#0f3d2e`, `--green-700:#1b6b45`, `--green-600:#1f7a4d`, `--green-500:#2e9e63` (акцент)
  - `--lime:#95c345` (primary-кнопки), `--seed:#f5ef2b`, `--wheat:#c9a227`
  - `--ink:#16241d`, `--muted:#5b6b62`, `--line:#e3ebe5`, `--bg:#f5f8f5`, `--card:#fff`
  - статусы: success `#2e9e63`, warning `#c9a227`/`#8a6d12`, error `#c0492f`
- **Карточки:** `--radius:16px`, мягкая тень `0 1px 2px ..., 0 8px 28px ...`, рамка `--line`.
- **Кнопки:** скруглённые 12px; primary — lime с тёмно-зелёным текстом; ghost/outline — зелёная рамка.

## Структура страницы (сверху вниз)

1. **Стики-шапка** (`header.nav`): логотип-лист SVG + «Прогресс Агро» + разделитель +
   подпись «PII-маскирование». Справа — health-dot (индикатор состояния) и блок API-ключа (скрытый до 403).
2. **Компактный hero**: глубокий зелёный градиент + листовой SVG-паттерн.
   - eyebrow «Защита персональных данных», заголовок «Маскирование персональных данных»,
     1–2 строки описания, бейджи-пилюли (RU/EN, «Данные не покидают сессию», TTL).
3. **Сессия-бар**: карточка — «Сессия {id}», кнопка «Сбросить», TTL-таймер.
4. **Баннер загрузки моделей** (warning-стиль `.note`), скрыт по умолчанию.
5. **Рабочая область** (2 колонки → 1 на мобильном): две белые карточки.
   - «Входной текст»: textarea на фоне `--bg`, моноширинный; футер — select RU/EN,
     чекбокс «Подсказка для ИИ», кнопка «Маскировать →» (lime).
   - «Результат»: contenteditable-блок; футер — «Копировать» (зелёный заливной), «← Восстановить» (outline).
6. **Сущности**: карточка-строка с фирменными чипами-пилюлями (цвет по типу сущности сохраняется).
7. **Статус-бар**: карточка в стиле `.info` со спиннером и текстом.
8. **Loading overlay**: на старте, перекрашен в светлую палитру с зелёным акцентом.

## Что НЕ меняется (инварианты)

- Весь `<script>`: id элементов, функции, эндпоинты (`BASE + 'mask'` и т.д.), localStorage-ключи,
  логика TTL/health/auth, обработчики paste/input, Ctrl+Enter.
- `ENTITY_COLORS` и подсветка по `masked_spans` через `Array.from` (code points).
- Контракт с API (поля запросов/ответов).
- Маршрут отдачи UI (`/` → `index.html`).

## Критерии готовности

- Файл открывается локально и на `http://localhost:6111/`, выглядит как корпоративный «Прогресс Агро».
- Все кнопки/поля работают: маскирование, восстановление, копирование, сброс, TTL, health-dot, API-ключ.
- Нет регрессий в JS (id-шники совпадают с тем, что ищет скрипт).
- Адаптив: рабочая область сворачивается в одну колонку на узких экранах.
- Без внешних зависимостей кроме шрифта Inter (как сейчас Roboto).
